# Тестовий кейс #116

```
Incident post-mortem · outage 20 травня · 47 хвилин

20 травня о 14:23 продукт ліг на 47 хвилин.

Що сталось: деплой нової версії AI suggestion engine спричинив memory leak.
За 10 хвилин сервер вичерпав RAM. Auto-scaling не спрацював вчасно.

Вплив: ~3,400 активних юзерів не могли зайти. 127 тікетів в підтримку.
3 Enterprise клієнти написали особисто.

Root cause: відсутність load testing перед деплоєм.
Contributing factors: тиск дедлайну, Влад деплоїв сам без second pair of eyes.

Що зробили:
→ Rollback за 7 хвилин після виявлення
→ Олена відповіла всім Enterprise клієнтам протягом 15 хвилин
→ Status page оновлювали кожні 10 хвилин

Зміни процесу:
→ Load testing: обов'язково перед кожним AI-related деплоєм
→ Деплой після 18:00 або в неробочий час (якщо не критично)
→ Two-person rule для production deployments
```
