# Тестовий кейс #38

```
Конфлікт в команді · ситуація між Катею і Романом

Отримала дві окремі скарги за один день — тригер для серйозної розмови.

Суть: Катя каже що Роман "ігнорує її дизайн без пояснень".
Роман каже що Катя "нав'язує рішення без розуміння технічних обмежень".

Мій аналіз: є реальна проблема в процесі передачі дизайну в розробку.
Немає єдиного місця де дизайнер пояснює рішення, а розробник — обмеження.
Це призводить до розмов "через мене".

Рішення-процес (не "хто правий"):
→Ввести design handoff template в Figma — Катя заповнює, Роман підтверджує
→ Weekly design-dev sync: 30 хвилин щосереди
→ "Право вето" з обов'язковим поясненням + альтернативою

Окремі розмови:
→ 1:1 з Катею: завтра 11:00
→ 1:1 з Романом: завтра 15:00
```
