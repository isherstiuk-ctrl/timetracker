# Тестовий кейс #69

```
Підготовка системи метрик для нового CSM

Customer Success Manager виходить 15 травня (якщо закриємо вакансію вчасно).

Що вони мають відстежувати:

Health score per account (побудувати до виходу):
→ Login frequency (Amplitude)
→ Feature adoption (скільки core features використовують)
→ NPS score (per account)
→ Support tickets count
→ Contract renewal date

Red flags (автоматичні нотифікації):
→ Не заходив > 7 днів
→ Support tickets > 3 за тиждень
→ NPS < 7

Quarterly Business Review (QBR) template:
→ Usage summary
→ Goals vs actuals
→ Feature requests logged
→ Renewal discussion

Технічно: Amplitude + Notion database + Slack нотифікації.
Побудувати до 10 травня щоб CSM зміг почати з готовою системою.
```
