# Тестовий кейс #102

```
Нотатки з книги "Thinking in Bets" Енні Дюк

Читаю вже 2 тижні. Ключові ідеї:

Глава 1: "Resulting" — оцінювати рішення за процесом, не за результатом.
Погане рішення може дати хороший результат (і навпаки).
→ Мій приклад: вирішила не йти в Notion. Якщо xTiles провалиться — чи це була погана рішення?
   Ні — якщо на момент вирішення у мене були хороші підстави.

Глава 4: "Wanna bet?" — ставити ставки на власні переконання тестує їх реальність.
→ Застосування: перед великими рішеннями питати "яка ймовірність що я права?"

Глава 7: Різниця між "outcome quality" і "decision quality".
Борд-зустріч пройшла добре — але це не означає що мої слайди були ідеальними.
Може просто повезло з настроєм борду.

Що беру в роботу:
→ Pre-mortem перед кожним великим рішенням: "як це може провалитись?"
→ Ведення decision log — записувати рішення + підстави + очікуваний результат
```
