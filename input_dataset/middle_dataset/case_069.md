# Процес ревью коду · покращення

Проблема: PR висять по 3–5 днів, блокують прогрес.
Правило 1: ревью протягом 24 годин в робочі дні — всі погодились.
Правило 2: не більше 400 рядків в одному PR — більше = розбити.
Правило 3: автор PR пояснює контекст в описі — що і чому змінили.
Інструменти: Linear для трекінгу, GitHub Actions для автоматичних перевірок.
Rotation: ревью по черзі, не завжди Ігор — розподілити навантаження.
Метрика: середній час до merge — зараз 4 дні, ціль 1 день.
