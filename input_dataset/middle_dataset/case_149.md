Записка

Автоматизувати деплой процес — Ігор почне в наступному спринті.
Поточний стан: деплой напівручний, займає 2+ години Ігоря щотижня.
Ціль: повністю автоматизований pipeline — push → test → staging → production.
GitHub Actions: для тестів вже є базова конфігурація — розширити.
Staging деплой: автоматично при мерджі в develop гілку.
Production деплой: тільки з main + manual approval від tech lead.
Rollback: автоматично якщо health check провалюється після деплою — це критично.
