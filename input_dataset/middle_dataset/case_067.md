# Checklist · перед деплоєм

Code review: мінімум 2 апруви для основних гілок, 1 для hotfix.
Тести: всі юніт і інтеграційні тести green, coverage не нижче 80%.
Staging: задеплоїти на staging і провести smoke тести вручну.
Backup: перевірити що є актуальний backup бази даних.
Rollback план: знати яку версію відкочувати і як це зробити швидко.
Моніторинг: перевірити що Datadog і Sentry налаштовані на нову версію.
Повідомлення: написати в #deployments в Slack до і після деплою.
