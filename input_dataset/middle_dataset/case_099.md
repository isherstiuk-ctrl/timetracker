# Налаштування CI/CD · план

Поточний стан: деплой напівручний, Ігор витрачає 2 год/тиждень.
Ціль: повністю автоматизований pipeline — push → test → deploy.
Кроки: налаштувати GitHub Actions для тестів (є шаблон), додати staging деплой.
Production деплой: тільки з main гілки після manual approval.
Rollback: автоматичний якщо health check провалюється після деплою.
Інструменти: GitHub Actions (вже використовуємо), AWS ECS для деплою.
Час на реалізацію: Ігор каже 3–4 дні — взяти в наступний спринт.
